AngularJS Filters

Posted on 2 June 2015 by Iain Plimmer

I've been doing rather alot of AngularJS in recent weeks and filters are really, really cool. Never want to hear the word Voldemort? Use a filter!

Why and when would I look to use a filter?

Filters are just piped into expressions in AngularJS, they return a function and can take as many parameters that you want to throw at them. They are mostly useful for formatting and replacement, but have other advanced usage when piggy-backed off directives.

A simple working example...

A good place to start is a string replacement filter. If you're not a muggle, you might want to avoid the word Voldemort, so let's make a filter to help with that.

{{ myVariable | ReplaceWord:'voldemort':'He who must not be named'}}

This filter will take three parameters, the original string, the word to 'censor' and the string to replace it with. In this filter we just take the sentence, convert it to an array and replace the offensive element. We could just do a replace, but this way we are replacing entire words, not just matching characters.

.filter('ReplaceWord', function () {
    return function (input, censor, replacement) {                    
        var words = input.split(' ');
        words.map(function (word, i) {
            if (word === censor) {                            
                words[i] = replacement;                   
            }            
        });
        return words.join(' ');
    }
})

Finally...

Filters might look trivial, and simple, but their power is that you can share them in their own module, or bundle as part of your program, and they're just ready to go. AngularJS has many pre-built filters for you to use, like date filters, number filters and array filters that are really useful, but the likelyhood is that you'll your own in no time.
